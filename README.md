# 理工校园导播系统

基于 Qt、FFmpeg、OpenCV、OpenGL 实现的校园导播系统，支持音视频采集、美颜处理、本地录制及网络推流全流程功能。

## 功能特性

- **多路视频源采集**：支持摄像头、屏幕、本地视频多路视频源
- **双音频源采集**：支持麦克风、桌面音频采集
- **视频叠加处理**：支持视频叠加缩放、层级调整
- **实时美颜功能**：基于 OpenCV 实现磨皮、美白实时美颜
- **硬件编解码**：集成 NVIDIA 显卡硬件编解码，提升处理效率
- **高性能渲染**：利用 OpenGL 实现 GPU 视频渲染
- **本地录制**：支持 MP4 格式本地录制
- **网络推流**：支持 RTMP/RTSP 双模式网络推流

## 技术栈

- **框架**：Qt 5.x / 6.x
- **音视频处理**：FFmpeg 6.0
- **计算机视觉**：OpenCV 4.0+
- **图形渲染**：OpenGL
- **平台**：Windows（可扩展至 Linux）

## 项目结构

```
videoCapture/
├── capture/          # 采集模块
├── clock/            # 时钟同步模块
├── decoder/          # 解码模块
├── demuxer/          # 解复用模块
├── encoder/          # 编码模块
├── event/            # 事件系统
├── filter/           # 滤镜模块
├── image/            # 图标资源
├── include/          # 头文件
├── muxer/            # 复用模块
├── renderer/         # 渲染模块
├── ui/               # 界面模块
```

## 核心功能模块

### 1. 音视频采集
- 支持 DirectShow（Windows）/ V4L2（Linux）视频采集
- 支持麦克风、桌面音频采集
- 多路视频源同步采集

### 2. 美颜处理
- 基于 OpenCV 的人脸检测算法
- 实时磨皮、美白效果
- 动态调节美颜参数

### 3. 硬件编码
- NVIDIA NVENC 硬件加速编码
- H.264 视频编码
- AAC 音频编码

### 4. 视频渲染
- OpenGL GPU 渲染
- 多路视频叠加
- 抗撕裂处理

### 5. 网络推流
- RTSP 推流（延迟 ~500ms）
- RTMP 推流（延迟 600~700ms）

## 构建说明

### 环境要求
- Qt 5.15+ 或 Qt 6.x
- FFmpeg 6.0 开发库
- OpenCV 4.0+ 开发库

### 编译步骤
1. 使用 Qt Creator 打开 `videoCapture.pro`
2. 配置项目构建环境
3. 编译并运行

## 使用说明

1. **选择视频源**：支持摄像头、屏幕、本地视频
2. **配置美颜参数**：调节磨皮、美白程度
3. **选择输出方式**：本地录制或网络推流
4. **开始推流/录制**：点击开始按钮

## 性能指标

- RTSP 推流延迟：约 500ms
- RTMP 推流延迟：约 600~700ms
- CPU 占用率：硬件编码时显著降低
